# Rejected candidates

Public log of tools the curator evaluated but did not list. Files are per atom (e.g. `keyword-research.md`, `on-page-seo-audit.md`). Each entry records: tool, date evaluated, verdict, one-phrase reason, and a "reconsider if" clause.

The point: make the editorial gate visible. If the curator rejects a project, the reason is on the record. A `reconsider if` clause that comes true is a legitimate PR ("this tool now meets the bar").

Contributors are welcome to open a PR that moves an entry from `rejected/<atom>.md` to `data/entries/<slug>.yaml` when the `reconsider if` condition has been met.
